## 所見

- **物差しとして機能している**: 較正床（易 = CO-6）はクリアし、天井（難 4 件中検出 1）に十分な弁別余地が残った。別モデル・資産変更後の比較に使える分布
- **強み**: 並行性（セッション共有・自己持続ループ）、認可欠落、並行配列、ルート登録順 — いずれも機構の完全解明つきで検出。design-checker 観点（既存規約との突合）が CO-5 検出に直結
- **弱点 → 資産改善候補**:
  1. **「diff 外の実行環境・適用条件」系が全滅圏**（PF-1 / CO-9 / CO-11、設定・ビルド 0/3）。レビュー観点に「実行環境前提（TZ・ランタイム・ビルドモード・マイグレーション適用条件）と設定・規約ファイルの突合」を明示追加する
  2. **fail-close 原則の明文化**: アクセス制御の既定値がフェイルオープンである設計を「良い後方互換」と追認した（CO-10）。認可系の既定値に fail-close を要求する観点を追加する
  3. 検出しても**重大度を低く付ける**傾向（CO-6/CO-8 = Warning、CO-12 = Info）。重大度較正は Critical 帯では一致しており、中位帯の問題
- 制限事項: 追加指摘の真偽判定（誤検知率）は未実施。2 agent 縮約は Full 構成より recall が低い可能性があり、本結果は「プロトコル v1 における下限」と解釈する
