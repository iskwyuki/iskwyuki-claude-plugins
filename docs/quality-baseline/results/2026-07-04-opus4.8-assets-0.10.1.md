## degradation bracket（Haiku 4.5・代表 3 ケース）

強モデル（Opus 4.8）に対する degradation の方向を安価に測るため、同一プロトコル（blind・2 agent 縮約・diff のみ・外部参照禁止）で **Haiku 4.5** を代表 3 ケースに適用した。母数が異なるため検出率は本走と直接比較せず、「Opus が完全検出したケースを弱モデルが保持できるか」で読む。

| case_id | 難易度 | 位置づけ | Opus 4.8 | Haiku 4.5 |
|---|---|---|---|---|
| CO-6 | 易 | 較正床（アンカー） | 検出（Warning） | 検出（low） |
| CO-2 | 中 | 中核（並行性・Opus 完全検出） | 検出（Critical） | 見逃し |
| CO-4 | 難 | 認可（Opus 完全検出） | 検出（Critical） | 見逃し |

- **較正床は弱モデルでも保持**: CO-6（易・情報露出）は Haiku でも検出。ただし 2 agent 中 1 のみが確信度 low で捕捉した辛勝であり、余裕は小さい。資産の最低保証は弱モデルでも辛うじて機能。
- **中核・難は弱モデルで脱落**: Opus が機構完全一致で Critical 検出した CO-2（並行実行へ渡る共有セッションの規約違反）を Haiku は見逃し、しかも一方の agent は当該の並行実行を「良い設計（パフォーマンス最適化）」と Good 評価して規約違反を看過した（自信ある偽陰性）。CO-4（認可分類ミス）も Haiku は表層の指摘（依存の未宣言・重複コード・ログ出力・定数配置）に流れ、核心の認可欠落に到達しなかった。
- **示唆**: 物差しはモデル弁別に機能している（Opus 完全検出 3/3 → Haiku 1/3）。弱モデルは「機構レベルの並行性・認可」で顕著に劣化し、表面的・機械的な指摘へ退行する。恒久運用で弱モデルを併用する場合、較正床は守れても中核バグの取りこぼしが増える前提で設計する必要がある。

## 所見

- **物差しは Opus でも機能**: 較正床（易 = CO-6）はクリア、天井（難 4 件中完全検出 1）に弁別余地が残る。**部分込み検出率は Fable と同率 76.9%** で、モデルを替えても分布が安定＝比較器として使える。
- **完全検出は Fable が僅かに上（CO-5 の差）**: Fable は「既存規約との突合」で diff 外の衝突ルートを推論し dead endpoint を検出したが、Opus はブラインド diff から衝突相手（diff 外の汎用ルート）を想起できず見逃した。逆に **難ケースでは Opus が部分理解で上回る**（CO-1 の無ロック read-modify-write を low で捕捉、Fable は完全見逃し）。「浅い規約突合は Fable、深い機構の部分把握は Opus」という質的な差。
- **Fable の弱点3点は Opus でも再現（＝資産側の課題であることが補強された）**:
  1. **実行環境・適用条件系は依然全滅圏**（PF-1／CO-9／CO-11、設定・ビルド 0/3・部分1）。diff 外のデプロイ設定・ランタイム前提・migration 適用条件との突合が、強モデルでも観点として与えられていないと発火しない。→ レビュー観点への「実行環境前提（TZ・ランタイム・ビルドモード・migration 適用条件）と設定/規約ファイルの突合」明示追加が引き続き有効。
  2. **fail-close 原則の未適用も再現**（CO-10）。Opus は「未設定＝全公開」の構造まで特定しながら「後方互換・実害なし」と好意的評価に倒した。→ 認可系の既定値に fail-close を要求する観点追加が引き続き有効。
  3. **重大度較正は Fable と逆方向にズレた**。Fable は中位帯を低く付ける傾向だったが、Opus は検出ケースを Critical に寄せる傾向（CO-2/CO-7/CO-8＝正解 Medium/High/Medium を Critical 評価）。一方 CO-12 は Info。正解 Critical の 2/2 一致は維持。中位帯の較正課題は「低く付ける（Fable）」から「高く付ける（Opus）」へ様相が変わり、モデル依存の較正特性として記録。
- **弱モデル degradation は方向が明確**（上記 bracket）: 較正床は保持されるが、中核の並行性・難の認可で弱モデルが脱落し表層指摘へ退行。Opus の強みが「機構の深い把握」にあることが裏側から確認された。
- **制限事項**: 追加指摘の真偽判定（誤検知率）は会社コード非取得のため未実施（run1 と同じ）。2 agent 縮約は Full 構成より recall が低い可能性があり、本結果は「プロトコル v1 における下限」と解釈する。degradation bracket は代表 3 ケースの方向確認に留め、弱モデルの全 13 ケース計測は未実施。
