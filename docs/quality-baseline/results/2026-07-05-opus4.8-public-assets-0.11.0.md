## degradation bracket（Haiku 4.5・代表 3 ケース）

強モデル（Opus 4.8）が **X/Y 両 agent で完全検出**した 3 ケースに、同一プロトコル（blind・2 agent 縮約・diff のみ・外部参照禁止）で Haiku 4.5 を適用。「強が完全検出したケースを弱モデルが保持できるか」で読む。

| case_id | 難易度 | 位置づけ | Opus 4.8 | Haiku 4.5 |
|---|---|---|---|---|
| PF-4 | 中 | ロジック（日付検証） | 検出（X Warning・Y Critical） | 検出（X Warning のみ。Y は regex を「正しい」と Good 評価＝偽陰性） |
| PF-5 | 中 | セキュリティ（RSC 秘密露出） | 検出（X/Y Critical） | 検出（X Critical のみ。**Y は「apiKey を props で渡す設計は環境分離の観点で良い」と脆弱性を Good 評価**＝自信ある偽陰性） |
| PF-8 | 中 | 設定・ビルド（standalone 突合） | 検出（X/Y Critical） | 検出（Y Critical のみ。X は Makefile の別問題に流れ standalone 突合を看過） |

- **弱モデルは公開 bracket 3/3 を「保持」したが、いずれも 2 agent 中 1 のみの辛勝**。他方の agent はしばしば**バグを Good 評価する自信ある偽陰性**に倒れた（PF-5 の秘密 prop 渡しを「良い設計」、PF-4 の形式のみ検証を「正しい」）。
- 4.1（会社 bracket）では Opus 完全検出 3/3 → Haiku 1/3 と大きく脱落したのに対し、本公開 bracket は 3/3 保持。差の理由は**ケースの性質**: 公開 bracket（client 秘密露出・standalone 設定・日付形式検証）は既知アンチパターン系で片肺でも拾えるのに対し、4.1 の脱落ケース（CO-2 並行セッション・CO-4 認可分類）は機構の深い把握を要した。
- **示唆**: 弱モデルは「既知パターン系」なら 2 agent 冗長で辛うじて保持できるが、個々の agent の信頼性は低く（半数が偽陰性・一部は明確に誤った Good 評価）、機構理解を要するケースでは 4.1 同様に脱落する前提で運用する。**2 agent 縮約の冗長性が弱モデルの取りこぼしを実際に救っている**ことが公開 bracket で確認された。

## 所見

- **公開のみ継続 recall の基準点を確立**: 会社ケース凍結後、公開 8 ケースで検出 3/8・部分込み 5/8。以後の公開再走はこの値を基準に「モデル交代・資産変更」の差を測る。**唯一の『難』ケース PF-2 が部分検出止まり・『易』ケース不在**という較正上の弱点は継続課題（[INDEX.md](../cases/INDEX.md) の較正床補充課題）。
- **設定・ビルドで質的な内訳が見えた**: 歴史的に 0/3 だったこのカテゴリで、**2 ファイル同時突合型（PF-8 Dockerfile↔next.config）は強モデルで Critical 検出**する一方、**diff 内に相方が見えない/意味的依存型（PF-1 の locale 列挙欠落・PF-6 の Makefile 仲間外れ）は見逃す**。「同一 diff 内に突合材料が両方あるか」が発火の分かれ目で、レビュー観点に「設定/ビルドファイル間の要求↔生成の突合（Dockerfile↔next.config、Makefile ターゲット間の依存連鎖、migration 適用条件）」を明示追加する改善余地が具体化した。
- **PF-7（実行時ミスマッチ）は強モデルでも未発火**: `timeoutMs: 180_000` を「マジックナンバー」としては見るが「この処理には短すぎる→abort→リトライ再実行」という実行時の意味的ミスマッチには到達しない。「宣言値が対象処理の実測特性に照らして妥当か（タイムアウト・バッファ・リトライ）」の観点が引き続き欠落。
- **追加指摘の質は高い（公開ゆえ検証可能）**: 会社走では誤検知率が未測定だったが、公開走ではスポット確認で追加指摘の多くが真の実装欠陥（stale closure・catch 握りつぶし・index=-1 誤返却・draft 漏洩）と確認でき、recall の高さが誤検知の氾濫を伴っていない兆候。次回以降、追加指摘の全件真偽判定（誤検知率）を公開コードで実測可能。
- **弱モデル degradation は「保持はするが片肺」**: 冗長な 2 agent が救う一方、個々は半数が偽陰性。恒久運用で弱モデルを併用する場合、2 agent 縮約の冗長性維持が最低保証の前提。
