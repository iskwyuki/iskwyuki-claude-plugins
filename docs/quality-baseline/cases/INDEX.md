# ケース索引（v1 — 2026-06-13 確定、全 13 件）

PF-* は公開ケース（本ディレクトリに詳細あり）。CO-* は非公開ケースで、詳細・SHA はローカル manifest（`~/dev/quality-baseline-private/cases.md`）のみが持つ。

| case_id | カテゴリ | 重大度 | 難易度 | 検出に必要な視点（一般化） |
|---|---|---|---|---|
| PF-1 | 設定・ビルド | High | 中 | 動的セグメントと generateStaticParams 返却形の突き合わせ |
| CO-1 | 非同期・競合 | High | 難 | read-modify-write 追加時の行ロック/状態ガード欠如と並行経路との交錯 |
| CO-2 | 非同期・競合 | Medium | 中 | 並行実行へ渡る共有 DB セッション（フレームワークの並行利用規約） |
| CO-3 | ロジック | Critical | 中 | 破壊的クリーンアップの「取得失敗・空結果＝全件削除」フェイルセーフ欠如 |
| CO-4 | セキュリティ・認可 | Critical | 難 | 大規模認証導入 diff の中の「書かれていない」所有権チェック |
| CO-5 | ロジック | Medium | 中 | 汎用パスパラメータルートと静的ルートの登録順（衝突相手は diff 外） |
| CO-6 | セキュリティ（情報露出） | Medium | 易 | エラー応答への例外詳細の直接埋め込み（CWE-209） |
| CO-7 | 状態管理 | High | 中 | effect 依存配列の不安定な関数 prop と再取得ループの組み合わせ |
| CO-8 | ロジック | Medium | 中 | 並行配列の片側のみフィルタによる index 対応の破れ |
| CO-9 | 設定・ビルド | High | 難 | 日付境界ロジックのランタイム TZ 前提（根拠が diff 外のデプロイ設定） |
| CO-10 | セキュリティ・認可 | High | 中 | 限定公開機能の「未設定＝全公開」フェイルオープン既定 |
| CO-11 | 設定・ビルド | High | 難 | migration journal の順序整合（適用条件の知識が必要） |
| CO-12 | hydration | Low | 中 | 同一スコープ内の SSR ガード不整合 |

## 分布

- カテゴリ: ロジック 3 / 設定・ビルド 3 / セキュリティ・認可系 3 / 非同期・競合 2 / 状態管理 1 / hydration 1
- 難易度: 易 1 / 中 8 / 難 4
- 重大度: Critical 2 / High 7 / Medium 3 / Low 1
- 「型・null」が 0 件なのは選定の結果（型チェッカーが機械検出する領域のため、本セットの趣旨である「人間/LLM レビューでしか捕まえられないバグ」に該当例が残らなかった）

## 採点運用

CO-6（易）は較正アンカー: これを見逃す構成は資産が壊れているとみなす。CO-4 / CO-9 / CO-11（難）はモデル・資産間の差を出す弁別ケース。
